------------------------------------------------
-
-　ＤＸフォントデータファイル作成ソフト説明書
-
-　Version 0.5
-
------------------------------------------------

・概要

　　　フォントの文字画像情報をファイルにする( ＤＸフォントデータファイルを作成する )ソフトです。

　　　CreateFontToHandle 関数ではパソコンにインストールされているフォントしか
　　使用することができないので、特定のフォントを間違いなく使用したい場合は
　　フォントファイルをソフトに同梱して、ソフト実行時に AddFontResourceEx などの
　　Win32API を使用して動的にフォントをインストールした上で CreateFontToHandle
　　を使用するといった方法を採る必要がありますが、このソフトを使用して作成した
　　ＤＸフォントデータファイルはＤＸライブラリの関数 LoadFontDataToHandle で
　　手軽に読み込むことができます。



・注意事項

　　　このソフトを使用して作成したＤＸフォントデータファイルをソフトに付属させる場合は
　　フォントファイルそのものを同梱する必要はなくなりますが、それでも各フォントの
　　ライセンスは有効ですので、このソフトを使用して作成したＤＸフォントデータファイルを
　　配布するソフトに付属させる場合もフォントファイル自体を同梱する場合と同様に
　　各フォントのライセンスに従ってください。



・利点

　　・CreateFontToHandle で作成したフォントハンドルより、LoadFontDataToHandle で
　　　ＤＸフォントデータファイルから作成したフォントハンドルのほうが高速に動作する

　　・小さい文字、少ない文字のみをＤＸフォントデータファイルに含めた場合は
　　　フォントファイルよりもファイルサイズが小さくなる


・欠点

　　・フォントファイルと異なり文字画像情報をビットマップとして保存しているので、
　　　使用したいサイズの数だけＤＸフォントデータファイルを作成する必要がある

　　・保存する文字のサイズ( 解像度 )に比例してＤＸフォントデータファイルの
　　　サイズが大きくなる



・ファイル説明

　　CreateDXFontData.exe
　　　　　ＤＸフォントデータファイルを作成するためのソフトです

　　Source
　　　　　CreateDXFontData.exe のプログラムソースが入っています

　　Sample
　　　　　CreateDXFontData.exe を使用してＤＸフォントデータファイルを作成する例となる
　　　　　バッチファイルやテキストファイルが入っています


・使い方

　　CreateDXFontData.exe はコマンドラインツールなので、以下のオプションを指定して実行します。

　　　CreateDXFontData.exe [/?][/Fフォント名][/Sサイズ][/T太さ][/Bビット数][/I][/C文字セット][/Aテキストファイルパス][/Oフォントデータファイルパス]

　　　　/?
　　　　　　ヘルプ表示オプション

　　　　　　　ソフトで使用できるオプションなどを表示するためのオプションです


　　　　/Fフォント名

　　　　　　フォント名指定オプション

　　　　　　　フォント名を指定するためのオプションです。
　　　　　　　CreateFontToHandle 関数の第一引数 FontName に対応しています。
　　　　　　　フォント名の途中にスペースが入る場合は "ＭＳ ゴシック" のように " で
　　　　　　　フォント名を括ります。


　　　　/Sサイズ

　　　　　　フォントサイズ指定オプション

　　　　　　　作成する文字画像の大きさを指定するためのオプションです。
　　　　　　　CreateFontToHandle 関数の第二引数 Size に対応しています。


　　　　/T太さ

　　　　　　フォントの太さ指定オプション

　　　　　　　作成する文字画像の文字の太さを指定するためのオプションです。
　　　　　　　CreateFontToHandle 関数の第三引数 Thick に対応しています。


　　　　/B１ピクセルのビット数

　　　　　　１ピクセルのビット数( 1 or 4 or 8 )指定オプション

　　　　　　　１ピクセル辺りのビット数を指定するためのオプションです。
　　　　　　　1 を指定するとアンチエイリアス無しのフォント画像になり、
　　　　　　　4 又は 8 を指定するとアンチエイリアス有りのフォント画像になります。
　　　　　　　ビット数が多いほど作成されるＤＸフォントデータファイルのサイズが
　　　　　　　大きくなります。
　　　　　　　4 より 8 の方がアンチエイリアスに使用される階調数が多くなり品質が
　　　　　　　良くなりますが、殆ど違いが分からない場合もあるので、ファイルサイズ
　　　　　　　のことを考慮するとアンチエイリアスを使用したい場合は 4 を指定したほうが
　　　　　　　良いかもしれません。


　　　　/I
　　　　　　イタリック指定オプション

　　　　　　　作成するＤＸフォントデータファイルのフォント画像をイタリック体にする場合に
　　　　　　　指定するオプションです。


　　　　/C文字セット名

　　　　　　文字セット指定オプション

　　　　　　　使用する文字セットを指定するためのオプションです。
　　　　　　　対応している文字セットは以下の５つです。

　　　　 　　　    DEFAULT  　　　　：デフォルト文字
　　　　  　　　   SHIFTJIS 　　　　：シフトJIS文字
　　　　 　　　    HANGEUL  　　　　：ハングル文字( Windows949, UHC )
　　　　  　　　   BIG5     　　　　：繁体文字
　　　　   　　　  GB2312   　　　　：簡体文字
　　　　   　　　  WINDOWS_1252	　　：欧文(ラテン文字の文字コードその１)
　　　　   　　　  ISO_IEC_8859_15　：欧文(ラテン文字の文字コードその２)
　　　　   　　　  UTF8 　　　　　　：UTF8


　　　　/Aテキストファイルパス

　　　　　　ＤＸフォントデータファイルに含める文字の一覧が載ったテキストファイルのパスを指定するためのオプション

　　　　　　　このオプションが指定されていない場合は CreateDXFontData.exe は指定の
　　　　　　　文字セットのすべての文字をＤＸフォントデータファイルに保存しようとしますが、
　　　　　　　その場合は使用しない文字も沢山ファイルに保存されてしまうので、ファイルサイズが
　　　　　　　非常に大きくなります。
　　　　　　　　なので、使用する文字だけを書き込んだテキストファイルを作成して、このオプションで
　　　　　　　そのテキストファイルを指定するとテキストファイルに書き込まれている文字の情報だけが
　　　　　　　ＤＸフォントデータファイルに保存されるのでファイルサイズを抑えることができます。
　　　　　　　( あとＤＸフォントデータファイル作成の時間も短縮することができます )
　　　　　　　　尚、このオプションで指定するテキストファイルの対応形式は
　　　　　　　「シフトJIS(メモ帳での名称は「ANSI」)」「UTF-16LE(メモ帳での名称は「Unicode」)」の２種類です。


　　　　/Oファイルパス

　　　　　　ＤＸフォントデータファイル保存ファイルパス指定用オプション

　　　　　　　作成するＤＸフォントデータファイルを保存するためのパスを指定するためのオプションです。



　　例：フォント「ＭＳ 明朝」のサイズ32、太さ1、アンチエイリアス付きの文字データを作成して、
　　　　ファイル名「MSMincho.dft」で保存する

	CreateDXFontData.exe /F"ＭＳ 明朝" /S32 /T1 /B4 -OMSMincho.dft


　　例：フォント「Arial Black」のサイズ16、太さデフォルト、アンチエイリアス無しの文字データを
　　　　テキストファイル「Char.txt」に書かれている文字のみ作成して、ファイル名「ArialBlack.dft」
　　　　で保存する

	CreateDXFontData.exe /F"Arial Black" /S16 /B1 /AChar.txt /OArialBlack.dft


　　上記の他に、「Sample」フォルダにＤＸフォントデータファイルを作成するバッチファイルのサンプルがあります。
　　実際にＤＸフォントデータファイルを作成する際は「Sample」フォルダ内のバッチファイルやテキストファイルを
　　改造して使ってみてください。



・ＤＸフォントデータファイルをプログラムで使用する方法

　　画像ファイルは LoadGraph関数で、サウンドファイルは LoadSoundMem関数で読み込むのと同じように
　　ＤＸフォントデータファイルは LoadFontDataToHandle という関数で読み込みます。

	FontHandle = LoadFontDataToHandle( "FontData.dft", 0 ) ;

　　　LoadFontDataToHandle の第一引数はＤＸフォントデータファイルのパスを、第二引数には縁の太さを
　　指定します。( 0 を指定すると縁無しフォントになります )
　　　LoadFontDataToHandle で取得したフォントハンドルは CreateFontToHandle と同じものですので、
　　DrawStringToHandle などを使用して描画を行い、不要になった際は DeleteFontToHandle で削除します。



・追加情報

　　・ＤＸＡファイル内に含まれているＤＸフォントデータファイルも LoadFontDataToHandle で読み込むことができます

　　・メモリに読み込んだＤＸフォントデータファイルからフォントハンドルを作成する関数 LoadFontDataFromMemToHandle があります
